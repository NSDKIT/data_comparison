# データ比較ツール 仕様書

## 概要

EP・EV・epori の3種類の計測データファイルをブラウザ上でインポートし、データ管理（サマリー）とグラフ表示を行うツール。

- **配布形式**: 単一 HTML ファイル（`index.html`）
- **動作環境**: インストール不要。ファイルをダブルクリックしてブラウザで開くだけで動作
- **対応ブラウザ**: Chrome / Edge / Firefox / Safari（iOS Safari 含む）
- **サーバー不要**: すべての処理はクライアントサイド（ブラウザ内）で完結

---

## ファイル構成

```
index.html          メインアプリ（xlsx.js・Chart.js・chartjs-adapter-date-fns を内包）
```

## インポートできるファイル

### data1（EP データ）

| 項目 | 内容 |
|------|------|
| 表示名 | data1 |
| 対応形式 | `.xlsx` / `.xls` |
| 内部キー | `ep` |
| データ構造 | 縦型（A列: タイムスタンプ、B列: 計測値） |
| ヘッダー | 1行目がヘッダー（文字列）の場合は自動スキップ |

### data2（EV データ）

| 項目 | 内容 |
|------|------|
| 表示名 | data2 |
| 対応形式 | `.xlsx` / `.xls` |
| 内部キー | `ev` |
| データ構造 | 縦型（A列: タイムスタンプ、B列: 計測値） |
| ヘッダー | 1行目がヘッダー（文字列）の場合は自動スキップ |

### data3（epori データ）

| 項目 | 内容 |
|------|------|
| 表示名 | data3 |
| 対応形式 | `.csv` / `.xlsx` |
| 内部キー | `epori` |
| データ構造 | 横型（1行 = 1日、列7以降に30分刻みの時刻ラベルと計測値） |
| 変換処理 | インポート時に縦型（タイムスタンプ × 計測値）へ自動変換 |

#### epori フォーマット詳細

```
列2 (index=2): 日付（YYYYMMDD 整数 / 文字列 / Excel Date オブジェクト）
列7〜       : 時刻ラベル（"0:00", "0:30", ... / Excel 時間値: 0, 0.020833, ...）と計測値
```

変換後の内部形式:

```
{ timestamp: "YYYY-MM-DDTHH:MM:00", value: number }
```

## 内部データ構造

すべてのファイルは以下の共通形式に正規化後、タイムスタンプ昇順でソートされる。

```js
[
  { timestamp: "2026-05-15T00:00:00", value: 123.4 },
  { timestamp: "2026-05-15T00:30:00", value: 125.0 },
  ...
]
```

## UI フロー

```
Step 1: ファイルインポート
  ├── data1 ドロップゾーン
  ├── VS 区切り
  └── data2 / data3 ドロップゾーン

       ↓ 2ファイル以上インポート完了

Step 2: 比較分析を開始（ボタン）

       ↓

Step 2: 分析結果セクション
  ├── タブ: データ管理
  └── タブ: グラフ
```

### ドロップゾーンの状態

| 状態 | 表示 |
|------|------|
| `idle` | アップロードアイコン・「ファイルをドロップ」 |
| `loading` | スピナー・「読み込み中...」 |
| `success` | ✓ アイコン・ファイル名・レコード数 |
| `error` | ✕ アイコン・エラーメッセージ |

### ファイル選択方法（iOS 対応）

- **デスクトップ**: ドラッグ＆ドロップ または クリックで選択
- **iOS Safari**: `<label>` で `<input type="file">` を包む実装により、タップでファイル選択ダイアログを起動

## 機能1: データ管理（データ管理タブ）

インポートされたデータのサマリーを表形式で表示。

| カラム | 内容 |
|--------|------|
| ファイル | data1 / data2 / data3 のバッジ |
| 開始 | 最小タイムスタンプ（YYYY-MM-DD HH:MM） |
| 終了 | 最大タイムスタンプ（YYYY-MM-DD HH:MM） |
| データ数 | 実際のレコード数 |
| 期待数 (30分刻み) | 開始〜終了を30分間隔で埋めた場合の理論値 |
| 欠損数 | 期待数 − データ数（負の場合は0） |

欠損数が1以上の場合はセルをオレンジ色で強調表示。

---

## 機能2: グラフ（グラフタブ）

インポートされた全データを折れ線グラフで重ね描き。

| 項目 | 設定 |
|------|------|
| ライブラリ | Chart.js 4.x（HTML 内に埋め込み済み） |
| 日付アダプター | chartjs-adapter-date-fns（HTML 内に埋め込み済み） |
| X 軸 | タイムスタンプ（type: `time`、Unix ミリ秒で渡す） |
| Y 軸 | 計測値 |
| 表示形式 | X軸: `M/d`〜`M/d HH:mm`（自動スケール）|
| インタラクション | ホバーで統合ツールチップ（`mode: 'index'`） |
| アニメーション | 無効（パフォーマンス優先） |
| 点の描画 | 無効（`pointRadius: 0`） |

### 系列の色

| データ | 色 |
|--------|----|
| data1 | `#3b82f6`（青） |
| data2 | `#10b981`（緑） |
| data3 | `#f59e0b`（オレンジ） |

## iOS Safari 対応

| 問題 | 対応 |
|------|------|
| `div.click()` でファイル選択が開かない | `<label>` + `<input type="file">` に変更 |
| `new Date("YYYY-MM-DDTHH:MM:SS")` が NaN | `parseTs()` で `-` → `/`、`T` → ` ` に変換してから `Date` を生成 |
| Chart.js 日付アダプターのパース失敗 | Unix ミリ秒（数値）を x 値として渡す |

## 外部依存ライブラリ（すべて HTML に埋め込み済み）

| ライブラリ | バージョン | 用途 |
|-----------|-----------|------|
| xlsx.js | 最新 | `.xlsx` / `.xls` / `.csv` の読み込み |
| Chart.js | 4.5.1 | グラフ描画 |
| chartjs-adapter-date-fns | 3.0.0 | Chart.js の時系列 X 軸 |

CDN は使用しない。すべてオフラインで動作する。

---

## 分析開始条件

2ファイル以上が `success` 状態になると「比較分析を開始」ボタンがアクティブになる。

---

## リセット

「リセット」ボタンで全ファイルの状態・パース済みデータ・分析結果セクションを初期状態に戻す。
